Проверка HMAC (Hash-based Message Authentication Code) — это метод защиты данных, который одновременно проверяет авторство сообщения и его целостность при взаимодействии между IT-сервисами. Этот способ гарантирует, что данные по пути не были изменены хакерами, а отправителем является именно тот сервис, за кого он себя выдает. [1, 2, 3] 
В основе метода лежит использование секретного ключа, известного только двум системам, и криптографической хэш-функции (например, SHA-256). [4, 5] 
------------------------------
## Как работает проверка HMAC (Пошагово)
Представьте, что Сервис А хочет отправить платежное поручение Сервису Б.

   1. Создание хэша: Сервис А берет текст сообщения и безопасный секретный ключ.
   2. Генерация подписи: С помощью алгоритма (например, HMAC-SHA256) Сервис А объединяет их и создает уникальную строку — сигнатуру (подпись). [6] 
   3. Отправка: Сервис А отправляет само сообщение и полученную сигнатуру по открытой сети. Секретный ключ никогда не передается.
   4. Проверка: Сервис Б получает сообщение и сигнатуру. Он берет это сообщение, свой экземпляр того же секретного ключа и снова запускает алгоритм HMAC. [7, 8] 
   5. Сравнение: Если вычисленная Сервисом Б подпись полностью совпадает с присланной — проверка пройдена успешно.
